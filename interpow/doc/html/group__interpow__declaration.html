Detailed Description

Macros for preliminary Task and Field declarations.

Channels
Channels allow two tasks to communicate with each other, or a task to communicate with itself. A channel is defined by its source and destination tasks, and is referred to as (source, destination).
Declarations
Declarations of tasks, initial task, fields and self-fields must be in the global area outside the main, and have to follow this order:
  • tasks declarations, using NewTask();
  • initial task declaration, using InitialTask();
  • fields and self-fields declarations, using NewField() and NewSelfField().

#define InitialTask (   TASK)

Define which task has to execute first on the very first start of the system. TASK has to be defined before using this macro.

Parameters
TASKname of the task
Non-volatility
The library requires the user to use a #pragma before calling this macro, in order to save the program state to non-volatile memory, i.e. 
#pragma PERSISTENT(PersState)
InitialTask(myInitialTask)
Note that the argument of PERSISTENT is fixed.

Define a new field, conceptually belonging to the channel (SRC, DST ). SRC and DST have to be defined before using this macro.

Parameters
SRCchannel's source task
DSTchannel's destination task
NAMEfield's name
TYPEfield's TYPE, must be a value of FIELD_TYPES
LENfield's length (if LEN>1 the field is an array)
Non-volatility
Fields have to be non-volatile, and have to be initialised only at the first start of the program. In order to do so, before declaring a field myField, belonging to the channel (ST, DT ), a #pragma has to be used, i.e. 
#pragma PERSISTENT(PersField(ST, DT, myField))
NewField(ST, DT, myField, type, len)

Define a new self-field, conceptually belonging to the self-channel (TASK, TASK ). TASK has to be defined before using this macro.

Parameters
TASKchannel's source and destination task
NAMEfield's name
TYPEfield's TYPE, must be a value of FIELD_TYPES
LENfield's length (if LEN>1 the field is an array)
CODEfield's code, must be a value of SELF_FIELD_CODES
Self-field code
A specific self-channel (T, T ) can have at most 8 self-fields. Self-fields belonging to the same self-channel must have different codes, but they do not need to be sequential.
Non-volatility
Self-fields have to be non-volatile, and have to be initialised only at the first start of the program. In order to do so, before declaring a self-field myField, belonging to the self-channel (T, T ), two #pragma have to be used, i.e. 
#pragma PERSISTENT(PersSField0(T, myField))
#pragma PERSISTENT(PersSField1(T, myField))
NewSelfField(T, myField, type, len, code)
